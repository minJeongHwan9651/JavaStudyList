# 📦 깃허브(GitHub) 필수 명령어 커닝페이퍼
이 문서는 터미널(까만 화면)에서 깃허브를 관리할 때 쓰는 **절대 까먹으면 안 되는 필수 명령어** 모음집입니다. 혼자 공부할 때는 복잡한 거 다 빼고 딱 이것만 씁니다!
---

## 🚀 매일 쓰는 마법의 3줄 (퇴근 전 필수)
오늘 공부(코딩, 마크다운 작성 등)를 모두 마쳤다면 터미널에 아래 3줄을 순서대로 칩니다.

### 1. 변경된 모든 파일 상자에 담기
```bash
git add .
```
주의: add와 . 사이에 무조건 띄어쓰기 한 칸이 있어야 합니다!

### 2. 상자 포장하고 메모 남기기 (이름표 붙이기)
```bash
git commit -m "오늘 공부한 내용 요약 적기"
```
예시: git commit -m "이것이 자바다 6장 객체지향 정리 및 코테 1문제"

### 3. 깃허브로 택배 보내기 (최종 전송)
```bash
git push
```
이 명령어를 쳐야 비로소 깃허브 사이트에 잔디가 심어지고 파일이 올라갑니다.

### 🚨 에러가 났을 때 (비상용 치트키)
깃허브 창고와 내 컴퓨터의 상태가 달라서 ! [rejected] 에러가 뜨면서 푸시(push)가 안 될 때 쓰는 강제 전송 명령어입니다. (나 혼자 쓰는 저장소에서만 쓸 것!)
```bash
git push -u origin main --force
```

### 🛠️ 처음 1번만 하는 초기 세팅 (새 폴더 연결할 때)
새로운 폴더를 만들고 깃허브의 빈 창고(Repository)와 처음 연결할 때 딱 한 번만 순서대로 입력합니다.

1. 깃 시작하기
```bash
git init
```

2. 깃허브 주소 연결하기([주소] 부분에 깃허브에서 복사한 .git 주소 넣기)
```bash
git remote add origin [주소]
```

3. 파일 담고 이름표 붙이기
```bash
git add .
git commit -m "첫 커밋: 폴더 초기 세팅"
```

4. 메인 가지(Branch) 이름 설정 및 최초 전송
```bash
git branch -M main
git push -u origin main
```
-u 옵션 덕분에 다음부터는 그냥 git push만 쳐도 알아서 날아갑니다!
