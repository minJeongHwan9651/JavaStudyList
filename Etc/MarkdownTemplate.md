# 📝 마크다운(Markdown) 필수 문법 커닝페이퍼

이 문서는 내가 깃허브에 블로그처럼 글을 쓸 때 참고하려고 만든 템플릿입니다.
자주 쓰는 핵심 기호들만 모아두었습니다.

---

## 1. 제목 달기 (Header)
`#` 개수에 따라 제목의 크기가 달라집니다. `#` 뒤에는 꼭 띄어쓰기를 해야 합니다.

# 가장 큰 제목 (보통 문서 제목에 1개만 씀)
## 두 번째로 큰 제목 (소주제 나눌 때)
### 세 번째로 큰 제목 (더 세부적인 내용)

---

## 2. 강조하기 (Text Formatting)
글씨에 포인트를 주고 싶을 때 사용합니다.

- **굵게 만들기**: 별표 2개로 감싸기 👉 **이것이 자바다 핵심 개념**
- *기울이기*: 별표 1개로 감싸기 👉 *시간 복잡도 주의*
- ~~취소선~~: 물결표 2개로 감싸기 👉 ~~이건 옛날 방식이라 안 씀~~

---

## 3. 목록 만들기 (Lists)
노트 필기할 때 가장 많이 쓰는 기호입니다.

### 순서가 없는 목록 (하이픈 `-` 사용)
- 코딩테스트 오답 체크
  - 스페이스바 2번 누르고 `-` 쓰면 이렇게 하위 목록이 됨
  - 인덱스 범위 초과 에러(ArrayIndexOutOfBounds) 조심할 것

### 순서가 있는 목록 (숫자와 점 `1.` 사용)
1. 문제를 꼼꼼히 읽는다.
2. 손으로 먼저 로직을 그려본다.
3. 코드를 작성한다.

---

## 4. 코드 작성하기 (Code Blocks)
개발자에게 가장 중요한 기능! 백틱(`)을 사용합니다.

### 인라인 코드 (문장 안에 짧은 코드나 변수명 넣기)
반복문을 돌릴 때 `int i = 0`부터 시작하는 것을 잊지 말자.

### 여러 줄의 코드 블록 (백틱 3개와 언어 이름)
```java
// 예쁘게 색칠된 자바 코드
public class Main {
    public static void main(String[] args) {
        System.out.println("이렇게 코드가 예쁘게 들어갑니다!");
    }
}
```
## 5. 인용구 쓰기 (Blockquote)
과외 선생님이 해주신 중요한 조언이나 책의 핵심 문장을 인용할 때 씁니다. (> 사용)
"코드를 무작정 짜기 전에, 시간 복잡도가 O(N)으로 끝날 수 있는지 먼저 고민해라!"
> 코테 과외 쌤

## 6. 체크리스트 만들기 (Task Lists)
오늘 할 일이나 진도표를 만들 때 씁니다.

[x] 깃허브 초기 세팅 완료하기
[x] 마크다운 필수 문법 테스트해보기
[ ] 이것이 자바다 다음 챕터 읽기
[ ] 코딩테스트 1문제 풀고 주석으로 오답 노트 남기
